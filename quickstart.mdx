## See It Work in 30 Seconds

Before building anything, see Faramesh in action with the built-in demo:

<Steps>
  <Step title="Install Faramesh">
    <CodeGroup>
    ```bash macOS / Linux (Homebrew)
    brew install faramesh/tap/faramesh
    ```

    ```bash macOS / Linux (curl)
    curl -fsSL https://raw.githubusercontent.com/faramesh/faramesh-core/main/install.sh | sh
    ```

    ```bash Docker
    docker pull ghcr.io/faramesh/faramesh:latest
    ```
    </CodeGroup>
  </Step>

  <Step title="Run the demo">
    ```bash
    faramesh demo
    ```

    You'll see live governance decisions in your terminal:

## Build Your First Governed Agent (5 minutes)

Now let's build a real governed agent from scratch.

<Steps>
  <Step title="Create a policy file">
    Create a new file called `payment.yaml`:

    ```yaml payment.yaml
    faramesh-version: '1.0'
    agent-id: payment-bot
    default_effect: permit

    vars:
      max_refund: 500

    rules:
      # Deny destructive shell commands — always.
      - id: deny-destructive-shell
        match:
          tool: shell/run
          when: 'args["cmd"] matches "rm\\s+-[rf]"'
        effect: deny
        reason: destructive shell command blocked by policy
        reason_code: DESTRUCTIVE_SHELL_COMMAND

      # Require human approval for high-value refunds.
      - id: defer-high-value-refund
        match:
          tool: stripe/refund
          when: 'args["amount"] > vars["max_refund"]'
        effect: defer
        reason: refund exceeds threshold — requires finance approval
        reason_code: HIGH_VALUE_REFUND

      # Deny mass email.
      - id: deny-mass-email
        match:
          tool: send_email
          when: 'len(args["recipients"]) > 50'
        effect: deny
        reason: mass email to >50 recipients requires manual review
        reason_code: MASS_EMAIL_LIMIT
    ```

    <Note>
      **Policy rules are evaluated in order.** First match wins. The `when` condition uses [expr-lang](https://github.com/expr-lang/expr) — a type-safe expression language that compiles to bytecode.
    </Note>
  </Step>

  <Step title="Validate the policy">
    Before using the policy, validate it to catch syntax errors:

    ```bash
    faramesh policy validate payment.yaml
    ```

    This command:
    - Parses the YAML structure
    - Validates rule definitions
    - Compiles all `when` conditions to bytecode
    - Exits 0 on success, 1 on error (perfect for CI pipelines)

    <Tip>
      Add this to your CI pipeline to prevent broken policies from reaching production.
    </Tip>
  </Step>

  <Step title="Install the Python SDK">
    ```bash
    pip install faramesh
    ```
  </Step>

  <Step title="Create your agent code">
    Create `agent.py`:

    ```python agent.py
    from faramesh import govern, DenyError, DeferredError

    # Define your agent functions
    def stripe_refund(amount: float, currency: str) -> dict:
        """Refund a charge via Stripe."""
        print(f"[STRIPE] Refunding {currency.upper()} {amount}")
        return {"status": "success", "amount": amount, "currency": currency}

    def send_email(subject: str, recipients: list[str]) -> dict:
        """Send an email."""
        print(f"[EMAIL] Sending to {len(recipients)} recipients: {subject}")
        return {"status": "sent", "count": len(recipients)}

    # Wrap with governance
    governed_refund = govern(
        stripe_refund,
        policy='payment.yaml',
        agent_id='payment-bot'
    )

    governed_email = govern(
        send_email,
        policy='payment.yaml',
        agent_id='payment-bot'
    )

    # Test the governed functions
    if __name__ == "__main__":
        # This will PERMIT (under threshold)
        try:
            result = governed_refund(amount=100, currency='usd')
            print(f"✓ Refund successful: {result}")
        except DenyError as e:
            print(f"✗ DENY: {e.reason}")
        except DeferredError as e:
            print(f"⏸ DEFER: {e.reason}")
            print(f"  Approve with: faramesh agent approve {e.defer_token}")

        # This will DEFER (exceeds threshold)
        try:
            result = governed_refund(amount=12000, currency='usd')
            print(f"✓ Refund successful: {result}")
        except DeferredError as e:
            print(f"⏸ DEFER: {e.reason}")
            print(f"  Approve with: faramesh agent approve {e.defer_token}")

        # This will DENY (too many recipients)
        try:
            recipients = [f"user{i}@example.com" for i in range(100)]
            result = governed_email(subject="Test", recipients=recipients)
            print(f"✓ Email sent: {result}")
        except DenyError as e:
            print(f"✗ DENY: {e.reason}")
    ```

    <Note>
      **`govern()` auto-starts the daemon on first call.** No separate setup required in development mode.
    </Note>
  </Step>

  <Step title="Run the agent">
    ```bash
    python agent.py
    ```

    You'll see:

    ```
    [STRIPE] Refunding USD 100
    ✓ Refund successful: {'status': 'success', 'amount': 100, 'currency': 'usd'}
    ⏸ DEFER: refund exceeds threshold — requires finance approval
      Approve with: faramesh agent approve def_abc123xyz
    ✗ DENY: mass email to >50 recipients requires manual review
    ```

    <Warning>
      The second refund (\$12,000) will **block** waiting for approval. Press `Ctrl+C` to cancel, or approve it in the next step.
    </Warning>
  </Step>

  <Step title="(Optional) Handle deferred actions">
    In a **separate terminal**, stream live decisions:

    ```bash
    faramesh audit tail
    ```

    You'll see real-time governance decisions as your agent runs.

    To approve a deferred action:

    ```bash
    faramesh agent approve def_abc123xyz
    ```

    To deny it:

    ```bash
    faramesh agent deny def_abc123xyz
    ```

    <Tip>
      In production, you'd integrate this with Slack, PagerDuty, or a custom approval workflow.
    </Tip>
  </Step>

  <Step title="Verify audit trail integrity">
    Every decision is recorded in a tamper-evident DPR (Decision-Provenance Record) chain:

    ```bash
    faramesh audit verify ~/.faramesh/faramesh.db
    ```

    Output:

    ```
    ✓ Chain integrity verified. 3 records, 0 violations.
    ```

    Each record includes:
    - SHA256 hash of this record
    - SHA256 hash of the previous record (linked list)
    - Policy version active at decision time
    - Effect (PERMIT/DENY/DEFER/SHADOW)
  </Step>
</Steps>

## What Just Happened?

<AccordionGroup>
  <Accordion title="1. govern() wrapped your function">
    The `govern()` decorator intercepts every function call **before** the underlying function executes. It:
    - Serializes the function arguments into a `CanonicalActionRequest`
    - Sends the request to the Faramesh daemon (auto-started on first call)
    - Waits for a governance decision (PERMIT/DENY/DEFER)
    - Raises an exception on DENY or DEFER timeout
    - Executes the original function on PERMIT
  </Accordion>

  <Accordion title="2. The daemon evaluated your policy">
    The Faramesh daemon loaded `payment.yaml`, compiled the policy rules to bytecode, and evaluated each rule in order:
    - **Rule 1 (deny-destructive-shell):** Doesn't match (tool is `stripe/refund`, not `shell/run`) → skip
    - **Rule 2 (defer-high-value-refund):** Matches! `args["amount"] > 500` → DEFER
    - No further rules evaluated (first match wins)
  </Accordion>

  <Accordion title="3. Decisions were recorded in the DPR chain">
    Before returning each decision to your agent, Faramesh wrote a record to the **Write-Ahead Log (WAL)** and then asynchronously replicated it to the SQLite DPR chain.

    This guarantees **no execution without an audit record** — even if the process crashes between the WAL write and the SQLite replication, the WAL replays on restart.
  </Accordion>

  <Accordion title="4. DEFER blocked waiting for approval">
    When a rule returns `effect: defer`, the Python SDK enters a **DeferPollLoop** — polling the daemon every 2 seconds to check if the action has been approved or denied.

    Default timeout: 5 minutes. After timeout, `DeferredError` is raised with `expired=True`.
  </Accordion>
</AccordionGroup>

## LangChain Integration

Faramesh preserves LangChain `@tool` decorator metadata:

```python
from langchain.tools import tool
from faramesh import govern

@tool
def stripe_refund(amount: float, currency: str) -> str:
    """Refund a charge via Stripe."""
    return f"Refunded {currency.upper()} {amount}"

govern_refund = govern(stripe_refund, policy='payment.yaml', agent_id='payment-bot')
# LangChain tool metadata (name, description, args_schema) is fully preserved.
```

## Common Commands

```bash
faramesh demo                              # See governance in action (30 seconds)
faramesh init                              # Auto-detect env, generate config
faramesh serve --policy policy.yaml        # Start the governance daemon
faramesh policy validate policy.yaml       # Validate and lint a policy file
faramesh policy inspect policy.yaml        # Show compiled policy summary
faramesh audit tail                        # Stream live decisions
faramesh audit verify faramesh.db          # Verify DPR chain SHA256 integrity
faramesh agent approve <defer-token>       # Approve a pending DEFER
faramesh agent deny <defer-token>          # Deny a pending DEFER
faramesh agent kill <agent-id>             # Activate kill switch for an agent
```
