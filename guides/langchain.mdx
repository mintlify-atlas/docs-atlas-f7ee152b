title: LangChain Integration
description: Add governance to LangChain agents and tools with full metadata preservation

Faramesh integrates seamlessly with LangChain, preserving all `@tool` decorator metadata including `name`, `description`, and `args_schema`. Govern your LangChain tools without breaking agent workflows.

## Quick Start

```python
from langchain.tools import tool
from faramesh import govern

@tool
def stripe_refund(amount: float, currency: str) -> str:
    """Refund a charge via Stripe. Use this when a customer requests a refund."""
    # Your Stripe API call
    return f"Refunded {amount} {currency}"

# Wrap the LangChain tool with governance
governed_refund = govern(
    stripe_refund,
    policy='payment-policy.yaml',
    agent_id='payment-agent'
)

# All LangChain metadata is preserved
print(governed_refund.name)         # "stripe_refund"
print(governed_refund.description)  # "Refund a charge via Stripe..."
print(governed_refund.args_schema)  # Pydantic schema
```

## Metadata Preservation

The `govern()` decorator preserves these LangChain attributes:

- **`name`**: Tool name used by the agent
- **`description`**: Natural language description for the LLM
- **`args_schema`**: Pydantic schema for argument validation
- **`__wrapped__`**: Reference to the original function

This ensures LangChain agents can continue to discover and call tools through the governance layer.

## ReAct Agent Example

Governance works transparently with LangChain agents:

```python
from langchain.agents import AgentExecutor, create_react_agent
from langchain.prompts import PromptTemplate
from langchain_openai import ChatOpenAI
from langchain.tools import tool
from faramesh import govern

# Define your tools with LangChain's @tool decorator
@tool
def search_database(query: str) -> str:
    """Search the customer database."""
    # Database query logic
    return f"Results for: {query}"

@tool
def send_email(to: str, subject: str, body: str) -> str:
    """Send an email to a recipient."""
    # Email sending logic
    return f"Email sent to {to}"

@tool
def refund_payment(order_id: str, amount: float) -> str:
    """Process a refund for an order."""
    # Refund processing logic
    return f"Refunded ${amount} for order {order_id}"

# Wrap all tools with governance
tools = [
    govern(search_database, policy='policy.yaml', agent_id='support-agent'),
    govern(send_email, policy='policy.yaml', agent_id='support-agent'),
    govern(refund_payment, policy='policy.yaml', agent_id='support-agent'),
]

# Create the agent (no changes to LangChain code)
llm = ChatOpenAI(model="gpt-4", temperature=0)
prompt = PromptTemplate.from_template(
    """Answer the following question:

    Question: {input}

    {agent_scratchpad}"""
)

agent = create_react_agent(llm, tools, prompt)
agent_executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# Run the agent — all tool calls are governed
result = agent_executor.invoke({
    "input": "Refund $500 to customer cust_123 and send them a confirmation email"
})
```

<Note>
  The agent's tool calls are evaluated **before** execution. If a refund exceeds your policy threshold, the agent receives a `DenyError` and can adjust its plan.
</Note>

## Policy for LangChain Tools

Write policies that match LangChain tool names:

```yaml policy.yaml
faramesh-version: '1.0'
agent-id: support-agent
default_effect: permit

vars:
  max_refund_without_approval: 1000
  max_email_recipients: 10

rules:
  # Require approval for high-value refunds
  - id: defer-high-refund
    match:
      tool: refund_payment
      when: 'args["amount"] > vars["max_refund_without_approval"]'
    effect: defer
    reason: refund exceeds $1000 threshold — requires manager approval
    reason_code: HIGH_VALUE_REFUND

  # Deny mass emails
  - id: deny-mass-email
    match:
      tool: send_email
      when: 'len(args["to"].split(",")) > vars["max_email_recipients"]'
    effect: deny
    reason: cannot send email to more than 10 recipients
    reason_code: MASS_EMAIL_BLOCKED

  # Deny database queries with SQL injection patterns
  - id: deny-sql-injection
    match:
      tool: search_database
      when: 'args["query"] matches "(?i)(drop|delete|truncate|update)\\s+table"'
    effect: deny
    reason: potential SQL injection detected
    reason_code: SQL_INJECTION_BLOCKED

  # Allow all other operations
  - id: allow-default
    match:
      tool: "*"
    effect: permit
```

## Structured Tools with Pydantic

LangChain's structured tools (using Pydantic for argument schemas) work seamlessly:

```python
from langchain.tools import StructuredTool
from pydantic import BaseModel, Field
from faramesh import govern

class RefundInput(BaseModel):
    order_id: str = Field(description="The order ID to refund")
    amount: float = Field(description="The refund amount in USD")
    reason: str = Field(description="Reason for the refund")

def refund_payment(order_id: str, amount: float, reason: str) -> str:
    """Process a refund for an order."""
    return f"Refunded ${amount} for order {order_id}: {reason}"

tool = StructuredTool.from_function(
    func=refund_payment,
    name="refund_payment",
    description="Process a refund for a customer order",
    args_schema=RefundInput,
)

# Add governance (args_schema is preserved)
governed_tool = govern(tool, policy='policy.yaml', agent_id='payment-agent')

# Use with LangChain agent
agent = create_react_agent(llm, tools=[governed_tool], prompt=prompt)
```

## Error Handling in Agents

Handle governance exceptions within your agent loop:

```python
from langchain.agents import AgentExecutor
from faramesh import DenyError, DeferredError
import logging

logger = logging.getLogger(__name__)

# Wrap agent execution with governance error handling
try:
    result = agent_executor.invoke({"input": "Process a $5000 refund"})
except DenyError as e:
    logger.warning(f"Tool call denied: {e.reason} (code: {e.reason_code})")
    # Option 1: Return error to user
    result = {"output": f"I cannot complete that action: {e.reason}"}
    # Option 2: Let agent retry with different parameters
    # (agent will see the exception and adjust its plan)
except DeferredError as e:
    logger.info(f"Tool call requires approval: {e.defer_token}")
    result = {
        "output": f"This action requires approval. Token: {e.defer_token}",
        "defer_token": e.defer_token
    }
```

## OpenAI Functions Agent

Governance also works with OpenAI Functions agents:

```python
from langchain.agents import create_openai_functions_agent
from langchain_openai import ChatOpenAI
from langchain.tools import tool
from faramesh import govern

@tool
def get_weather(location: str) -> str:
    """Get the current weather for a location."""
    return f"Weather in {location}: 72°F, sunny"

@tool
def book_flight(origin: str, destination: str, date: str) -> str:
    """Book a flight."""
    return f"Booked flight from {origin} to {destination} on {date}"

# Add governance

    govern(get_weather, policy='policy.yaml'),
    govern(book_flight, policy='policy.yaml'),

llm = ChatOpenAI(model="gpt-4", temperature=0)

```

## Tool Chaining with LCEL

Use governed tools in LangChain Expression Language chains:

```python
from langchain_core.runnables import RunnableLambda
from langchain.tools import tool
from faramesh import govern

@tool
def fetch_customer(customer_id: str) -> dict:
    """Fetch customer data from the database."""
    return {"id": customer_id, "name": "Alice", "email": "alice@example.com"}

@tool
def send_notification(customer_data: dict, message: str) -> str:
    """Send a notification to a customer."""
    return f"Sent '{message}' to {customer_data['email']}"

# Add governance
governed_fetch = govern(fetch_customer, policy='policy.yaml')
governed_send = govern(send_notification, policy='policy.yaml')

# Chain governed tools
chain = (
    RunnableLambda(lambda x: governed_fetch.invoke({"customer_id": x["customer_id"]}))
    | RunnableLambda(lambda customer: governed_send.invoke({
        "customer_data": customer,
        "message": "Your order has shipped!"
    }))
)

result = chain.invoke({"customer_id": "cust_123"})
```

## Observability

Monitor governed LangChain agents in real-time:

```bash
# Stream live governance decisions
faramesh audit tail

# Output:
# [10:00:15] PERMIT  search_database    query="customer orders"       latency=8ms
# [10:00:17] DENY    send_email         recipients=847                policy=deny-mass-email  latency=2ms
# [10:00:20] DEFER   refund_payment     amount=$5000                  awaiting approval  token=abc123
```

Each tool call includes:
- **Tool name**: The LangChain tool's `name` attribute
- **Arguments**: Serialized call arguments (structural signatures, never raw values)
- **Decision**: PERMIT, DENY, DEFER, or SHADOW
- **Latency**: Governance evaluation time
- **Rule ID**: Which policy rule matched

<Tip>
  Use `faramesh audit verify` to verify the integrity of your DPR audit chain:
  ```bash
  faramesh audit verify ~/.faramesh/faramesh.db
  # ✓ Chain integrity verified. 1,247 records, 0 violations.
  ```
</Tip>

## Best Practices

<AccordionGroup>
  <Accordion title="Use descriptive tool IDs">
    Set explicit `tool_id` values for better policy readability:
    ```python
    governed_tool = govern(
        my_langchain_tool,
        policy='policy.yaml',
        tool_id='stripe.refund'  # Explicit, hierarchical ID
    )
    ```
  </Accordion>

  <Accordion title="Share agent_id across tools">
    Group related tools under the same `agent_id` for session tracking:
    ```python
    tools = [
        govern(search_db, policy='policy.yaml', agent_id='support-agent'),
        govern(send_email, policy='policy.yaml', agent_id='support-agent'),
        govern(refund, policy='policy.yaml', agent_id='support-agent'),
    ]
    ```
  </Accordion>

  <Accordion title="Test policies with faramesh demo">
    Use the demo mode to test your policy before connecting real tools:
    ```bash
    faramesh demo --policy policy.yaml
    ```
  </Accordion>

  <Accordion title="Handle DEFER gracefully">
    Design your agent to communicate approval requirements to users:
    ```python
    try:
        result = governed_tool.invoke(args)
    except DeferredError as e:
        return f"This action requires approval. Please contact your manager with token: {e.defer_token}"
    ```
  </Accordion>
</AccordionGroup>
